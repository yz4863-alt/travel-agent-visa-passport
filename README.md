# AI Travel Agent

AI Travel Agent is a static web app that visualizes where a selected passport can travel. It includes generated Wikipedia-sourced datasets for all 193 UN member-state passports plus Palestine, renders a local GeoJSON world map, and highlights each country by access category:

- Blue: visa-free
- Green: eVisa
- Light green: visa on arrival
- Gray: visa required or unknown
- Amber outline plus the access color: additional conditional access based on a selected US, UK, Schengen, or Canada visa

The interface is deployable as static files on Netlify, GitHub Pages, Cloudflare Pages, or any static host.

## How It Works

The browser loads:

- `data/world/countries.geojson` for country shapes
- `data/passports/index.json` for all configured passports
- `data/passports/<slug>.json` for Wikipedia-sourced visa records
- `assets/flags/*.svg` for local flag images in the selector and detail cards
- `src/background.js` for the Three.js animated AI-network background

The frontend in `src/app.js` matches map countries against the selected passport dataset, colors the SVG map, and shows hover/click details from the JSON record. The map supports smooth wheel/button zooming and drag panning.

## Data Structure

Each passport file follows this shape:

```json
{
  "passport": "United States",
  "source": "Wikipedia",
  "lastUpdated": "2026-07-02",
  "countries": {
    "France": {
      "status": "visa-free",
      "requirement": "Visa not required",
      "stay": "90 days",
      "notes": "90 days within any 180-day period in the Schengen Area."
    }
  },
  "visaEnhancements": {
    "us": {
      "label": "US visa",
      "countries": {}
    }
  }
}
```

Country names for Hong Kong, Macau, and Taiwan are normalized as `Hong Kong SAR, China`, `Macau SAR, China`, and `Taiwan (Province of China)`.

## Updating Wikipedia Data

Visa data is generated by `scripts/updateVisaData.js`. It fetches Wikipedia pages such as:

- `Visa requirements for United States citizens`
- `Visa requirements for Emirati citizens`
- `Visa requirements for Indian citizens`
- `Visa requirements for Philippine citizens`
- `Visa requirements for Santomean citizens`

Run:

```bash
npm run update:data
```

The full refresh targets all 193 UN member states plus Palestine. Wikipedia may rate-limit large runs, so the script includes request throttling and retry handling. If a run is interrupted or rate-limited, resume without refetching completed passport files:

```bash
npm run update:data -- --resume
```

To slow requests further:

```bash
WIKI_DELAY_MS=1500 npm run update:data -- --resume
```

To refresh one configured passport:

```bash
npm run update:data -- india
```

The parser reads Wikipedia visa requirement tables, maps requirements into the app categories, and writes refreshed JSON files into `data/passports`. Because Wikipedia table formats can change, review the generated JSON after major updates.

The configured passport list lives in `scripts/unMemberStates.js`. To change a demonym or add a search alias for a Wikipedia page title, edit that file.

## Refresh Button

The page includes a refresh button beside the Wikipedia data timestamp. In the default static build, the button explains the owner workflow because a browser cannot safely rewrite JSON files or redeploy a static site by itself.

For a real one-click production refresh, connect the frontend to a protected backend endpoint:

```html
<script>
  window.AI_TRAVEL_AGENT_REFRESH_ENDPOINT = "https://your-domain.example/api/refresh-visa-data";
</script>
```

That endpoint should authenticate the request, run `npm run update:data -- --resume`, validate with `npm run build`, commit or upload the regenerated `data/passports/*.json` files, and trigger a static-site redeploy. Keep this endpoint private or protected so public visitors cannot repeatedly trigger expensive Wikipedia refresh jobs.

## Local Development

Install Node.js 18 or newer, then run:

```bash
npm run dev
```

Open the printed local URL. The static server is only for local development; the app itself has no backend.

Validate the project:

```bash
npm run build
```

Validation checks that all configured passports are present, that each passport file has usable status values, and that Hong Kong, Macau, and Taiwan follow the required names.

Flag icons are bundled as static SVG files under `assets/flags`. If you add new country-code mappings in `src/countryFlags.js`, add the matching lowercase ISO-style SVG file to that folder.

## Deployment

Deploy the repository root as a static site.

For Netlify:

- Build command: `npm run build`
- Publish directory: `.`

For GitHub Pages:

- Commit the repository
- Enable Pages from the branch root
- Keep the `assets`, `data`, `src`, `index.html`, and `README.md` files in the published branch

No server-side scraping is required at runtime. Refresh the JSON data locally with `npm run update:data`, commit the changed files, and redeploy.
